# Agent Rules

Always read `docs/repo-context.md` and `docs/feature-map.md` first.

When implementing a feature:

1. Identify which files need modification.
2. Modify minimal files.
3. Avoid scanning the entire repository.

Update `repo-context.md` ONLY if architecture changes.

Update `feature-map.md` ONLY if product functionality changes (new features, removed features, or changed user flows).
